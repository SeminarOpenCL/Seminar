Detailed Description

Wrapper for cl_context.

A Context can be set up for one or several Device objects in the same Platform with several command Queue objects attached to Device objects. There might several Context objects for a Platoform. A Context is responsible for Kernel, Program, Event and Memory objects. Manipulation on these are performed using command Queue objects.
Kernel, Program, Event and Memory objects do insert themselves into a Context when their constructed either by taking the current active Context which has to be set by the user or when it is provided in their argument list.

Note that there can only be one active Context within one Platform.
