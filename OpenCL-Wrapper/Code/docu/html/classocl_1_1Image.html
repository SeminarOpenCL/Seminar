ocl::Image::copyTo Copies from this Image to the destination Image.

The operation assumes that all data are valid and no synchronization is necessary (active Queue executes in-order). The operation forces that all commands within the active Queue including this one are completed.

Parameters
src_originis the 3D offset in bytes from which the Image is read.
regionis the 3D region of the data. It is given with {image_width, image_height, image_depth}.
destis the Image into which the data is going to be copied.
dest_originis the 3D offset in bytes from which the destionation Image is read.

const size_t *  region,

ocl::Image::map Maps the Image into the host memory.

No data transfer is performed. Note that in order to map data of the Image the active queue must be a cpu and must have been allocated with the Image access mode AllocHost. You cannot modify the Image with OpenCL until unmap.

Parameters
originis the 3D offset in bytes from which the image is read.
regionis the 3D region in bytes to be mapped.
accessspecifies in what way the host_mem is used.
Returns
a void pointer to the mapped host memory location.
